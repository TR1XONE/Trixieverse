# TRIXIEVERSE: DEPLOYMENT OCH LANSERING\n\nDenna guide beskriver den tekniska processen för att driftsätta TrixieVerse-plattformen permanent via Railway.\n\n---\n\n## FAS 1: KONTO OCH INFRASTRUKTUR\n\n| STEG | ÅTGÄRD | INSTRUKTION |\n| :--- | :--- | :--- |\n| 1.1 | Skapa konto | Gå till [railway.app](https://railway.app) och logga in med ditt GitHub-konto. |\n| 1.2 | Initiera projekt | Klicka på **+ New Project** och välj **Deploy from GitHub repo**. |\n| 1.3 | Välj repository | Sök efter `Trixieverse` och klicka på **Deploy Now**. |\n| 1.4 | Provisionera databas | Klicka på **+ Add Service**, välj **Database** och sedan **PostgreSQL**. |\n\n---\n\n## FAS 2: KONFIGURATION (MILJÖVARIABLER)\n\n*Gå till din app-service (Trixieverse) och välj fliken **Variables**.*\n\n| VARIABEL | KÄLLA / VÄRDE | SYFTE |\n| :--- | :--- | :--- |\n| `DATABASE_URL` | `${{Postgres.DATABASE_URL}}` | Kopplar appen till din PostgreSQL-databas. |\n| `JWT_SECRET` | Valfri unik sträng | Kryptering av användarsessioner. |\n| `NODE_ENV` | `production` | Optimerar appen för driftmiljö. |\n| `PORT` | `3000` | Definierar nätverksport för trafik. |\n\n---\n\n## FAS 3: NÄTVERK OCH PUBLICERING\n\n| STEG | ÅTGÄRD | INSTRUKTION |\n| :--- | :--- | :--- |\n| 3.1 | Generera domän | Gå till fliken **Settings** -> **Networking** -> Klicka på **Generate Domain**. |\n| 3.2 | Verifiera status | Vänta tills deployment-loggen visar \"Success\" och status är \"Active\". |\n| 3.3 | Slutkontroll | Besök den genererade URL:en för att säkerställa att plattformen svarar korrekt. |\n\n---\n\n## TEKNISK SPECIFIKATION\n\n- **Runtime:** Node.js (LTS)\n- **Databas:** PostgreSQL 15+\n- **Build Tool:** Vite / NPM\n- **Protokoll:** HTTPS (Automatisk SSL via Railway)\n\n---\n\n## UNDERHÅLL OCH UPPDATERINGAR\n\n- **Automatisk Deployment:** Varje `git push` till `main`-branchen på GitHub triggar en ny build och deployment.\n- **Loggning:** Realtidsloggar finns tillgängliga under fliken **Logs** i Railway-dashboarden.\n- **Skalning:** Vid behov kan CPU och RAM justeras under fliken **Settings**.\n
